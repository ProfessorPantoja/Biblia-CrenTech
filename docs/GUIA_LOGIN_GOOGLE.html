🔑 Guia: Login com Google

Configuração que você precisa fazer (uma vez) para o login funcionar. ~15–20 min.
Projeto Supabase: gusysdjcmdzfqaaehdkr · Abordagem: Supabase Auth + Google
Como funciona: o Google cuida da identidade, e o Supabase fica no meio gerenciando a sessão e guardando os dados. Você vai (1) criar uma credencial no Google, (2) colar essa credencial no Supabase, e (3) rodar um trecho de SQL. Só isso.

Parte 1 — Google Cloud (criar a credencial OAuth)

1 Criar/abrir um projeto

  1. Acesse console.cloud.google.com com sua conta Google.
  2. No topo, crie um projeto novo (ex.: biblia-crentech) ou use um existente.

2 Tela de consentimento (OAuth consent screen)

  1. Menu → APIs e Serviços → Tela de consentimento OAuth.
  2. Tipo de usuário: Externo → Criar.
  3. Preencha o básico: nome do app (Bíblia CrenTech), e-mail de suporte (o seu) e e-mail de contato do desenvolvedor (o seu). Salve.
  4. Em Usuários de teste, adicione o seu próprio e-mail (enquanto o app estiver "em teste", só e-mails listados conseguem logar — depois você publica).

3 Criar as credenciais (Client ID)

  1. Menu → APIs e Serviços → Credenciais → Criar credenciais → ID do cliente OAuth.
  2. Tipo de aplicativo: Aplicativo da Web.
  3. Em Origens JavaScript autorizadas, adicione:
    http://localhost:3000
    https://biblia-crentech.vercel.app
  4. Em URIs de redirecionamento autorizados, adicione o callback do Supabase:
    https://gusysdjcmdzfqaaehdkr.supabase.co/auth/v1/callback
  5. Clique em Criar. O Google vai mostrar um Client ID e um Client Secret — copie os dois (vamos usar na Parte 2).
⚠️ Guarde o Client Secret com cuidado e não coloque no código/git. Ele vai só dentro do Supabase.

Parte 2 — Supabase (ativar o provedor Google)

4 Ligar o Google no Supabase

  1. Acesse o painel do seu projeto → Authentication → Providers.
  2. Encontre Google e ative (Enable).
  3. Cole o Client ID e o Client Secret que você copiou do Google. Salve.

5 Liberar os endereços de retorno

  1. Vá em Authentication → URL Configuration.
  2. Em Site URL, coloque:
    https://biblia-crentech.vercel.app
  3. Em Redirect URLs, adicione também o ambiente local:
    http://localhost:3000
  4. Salve.

Parte 3 — Banco de dados (para a sincronização)

6 Criar a tabela de dados do usuário

  1. No painel do Supabase, vá em SQL Editor → New query.
  2. Cole o SQL abaixo e clique em Run. Ele cria a tabela e a segurança (cada pessoa só acessa os próprios dados).
-- Tabela com os dados sincronizados de cada usuário
create table if not exists public.user_data (
  user_id uuid primary key references auth.users(id) on delete cascade,
  data jsonb not null default '{}'::jsonb,
  updated_at timestamptz not null default now()
);

-- Liga a segurança por linha (RLS)
alter table public.user_data enable row level security;

-- Cada usuário só pode ler/gravar a PRÓPRIA linha
create policy "own_data_select" on public.user_data
  for select using (auth.uid() = user_id);
create policy "own_data_insert" on public.user_data
  for insert with check (auth.uid() = user_id);
create policy "own_data_update" on public.user_data
  for update using (auth.uid() = user_id);
