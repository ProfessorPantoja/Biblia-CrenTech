🎯 Resumo executivo

Os dois becos sem saída do app acabaram: os cartões Histórico e Quiz da Home levavam a telas "Em Construção 🚧". Agora ambos funcionam de verdade.

De quebra, foi encontrado e corrigido um bug crítico de persistência que apagava seus dados a cada recarregamento — era ele o culpado da barra de histórico "sumida" (reclamação antiga registrada no FEATURES_APROVADAS).

🐛 Bug crítico corrigido

FixDados apagados ao recarregar o app · cc3a173

Sintoma: histórico, favoritos e última leitura se perdiam a cada reload. A barra lateral de histórico parecia "desativada" por isso.

Causa: o app carregava os dados salvos num useEffect, mas o efeito que salva rodava antes, com o estado ainda vazio, e sobrescrevia o localStorage.

Correção: os dados agora são lidos de forma síncrona na inicialização do estado — a corrida deixou de existir. Verificado no navegador: dados sobrevivem ao reload.

Reverter só isto: git revert cc3a173 (não recomendado — os dados voltam a sumir)
